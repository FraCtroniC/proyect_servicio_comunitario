# Guía para contribuir

Gracias por tu interés en contribuir a este proyecto. Sigue estas pautas para facilitar la revisión y la integración de tus cambios.

1. Fork -> Branch -> PR
   - Haz fork del repositorio y crea una rama con un nombre descriptivo (`feat/login`, `fix/attendance-bug`).
2. Estilo de commits
   - Usa mensajes claros; se recomienda Conventional Commits (`feat:`, `fix:`, `chore:`).
3. Tests
   - Añade o actualiza tests cuando añadas funcionalidad o arregles bugs.
   - Ejecuta `npm run test` y asegúrate de que todo pase.
4. Linter y formateo
   - Ejecuta `npm run lint` y `npm run format` si están disponibles.
5. Pull Request
   - Abre un PR hacia la rama `main` (o la rama establecida por el mantenedor).
   - Describe el cambio, el motivo y los pasos para probarlo.

Si tienes dudas, abre un issue para discutir la propuesta antes de implementar cambios grandes.
